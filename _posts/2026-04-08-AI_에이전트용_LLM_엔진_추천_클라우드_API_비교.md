## 모델별 상세 분석

### Anthropic — Claude 시리즈

#### Claude Opus 4.6 — 에이전트 종합 1위

| 항목 | 내용 |
|------|------|
| 가격 | 입력 $5 / 출력 $25 (1M 토큰) |
| 컨텍스트 | **1M 토큰** (장문 할증 없음) |
| SWE-bench | **80.8%** (코딩 에이전트 최상위) |
| TAU2-bench | 텔레콤 **99.3%**, 리테일 **91.9%** (1위) |
| LMSYS Arena | **종합 1위** |

**에이전트 강점**:
- 시스템 프롬프트 준수력이 가장 높음 → 복잡한 에이전트 지시에 정확하게 따름
- TAU-bench(실제 고객 서비스 시뮬레이션)에서 압도적 1위
- 멀티스텝 추론 + 도구 호출을 동시에 처리하는 능력 최고

**적합한 용도**: 고정밀 에이전트, 복잡한 멀티스텝 워크플로우, 코딩 에이전트

#### Claude Sonnet 4.6 — 가성비의 제왕

| 항목 | 내용 |
|------|------|
| 가격 | 입력 $3 / 출력 $15 (1M 토큰) |
| 컨텍스트 | **1M 토큰** |
| SWE-bench | **79.6%** |

**왜 에이전트에 최적인가**:
- Opus의 약 **60% 가격**으로 **98% 수준의 성능**
- 1M 컨텍스트를 표준 요금으로 사용 가능 (2026년 장문 할증 제거)
- OpenClaw, Hermes Agent 커뮤니티에서 **가장 많이 추천되는 모델**

**적합한 용도**: 일상적인 에이전트 작업의 메인 엔진, 코드 생성, 문서 분석

#### Claude Haiku 4.5 — 라우터 + 단순 작업

| 항목 | 내용 |
|------|------|
| 가격 | 입력 $1 / 출력 $5 (1M 토큰) |

**적합한 용도**: 작업 분류(라우터), 간단한 QA, 데이터 추출, 요약

### OpenAI — GPT 시리즈

#### GPT-4o / GPT-5.4 — 도구 호출 안정성 최고

| 항목 | 내용 |
|------|------|
| 가격 | 입력 $2.50 / 출력 $10 (1M 토큰) |
| 컨텍스트 | 128K 토큰 |
| SWE-bench | ~80% (GPT-5.2 기준) |

**에이전트 강점**:
- **도구 호출(Function Calling) 안정성이 업계 최고** — JSON 스키마 정확도가 가장 높음
- 가장 넓은 생태계와 서드파티 통합
- 프로토타이핑 → 프로덕션 전환이 매끄러움

**약점**: 컨텍스트 윈도우가 128K로 Claude·Gemini 대비 제한적

#### o4-mini — 추론형 저비용 모델

| 항목 | 내용 |
|------|------|
| 가격 | 입력 $0.55 / 출력 $2.20 (1M 토큰) |
| 컨텍스트 | 200K 토큰 |

**주의**: "추론 토큰"이 별도로 과금되어 실제 비용이 표면 가격보다 높을 수 있음. 단순 도구 호출에는 GPT-4o-mini가 더 경제적.

#### o3 — 고난도 추론 전용

| 항목 | 내용 |
|------|------|
| 가격 | 입력 $10 / 출력 $40 (1M 토큰) |

수학·과학·복잡한 논리 추론에서 압도적이나, 일반 에이전트 작업에는 과도한 비용. **에스컬레이션 전용**으로만 사용 권장.

## 비용 최적화 전략 — 티어별 모델 라우팅

모든 작업에 비싼 모델을 쓸 필요는 없습니다. **작업 난이도에 따라 모델을 자동 분배**하면 60~80% 비용을 절감할 수 있습니다.

### 3단계 라우팅 전략

| 티어 | 모델 | 용도 | 비중 |
|------|------|------|------|
| **Tier 1** (단순) | Haiku 4.5 / GPT-4o-mini | 분류, 추출, 간단 QA, 라우팅 | 60% |

| **Tier 3** (복잡) | Opus 4.6 / o3 | 멀티스텝 추론, 아키텍처 설계 | 10% |

### 추가 절감 팁

### API 안정성

프론티어 모델은 가끔 성능이 변동(regression)될 수 있습니다. 에이전트를 프로덕션에 배포할 때는:
- 특정 모델 버전을 **고정(pin)**할 것
- **폴백(fallback) 모델**을 반드시 설정할 것
- 주기적으로 벤치마크 테스트를 돌릴 것

## 마무리

2026년 4월 기준, AI 에이전트용 LLM의 선택지는 그 어느 때보다 풍부합니다.

**한 줄 요약**:
- **품질 우선** → Claude Sonnet 4.6 (메인) + Opus 4.6 (에스컬레이션)
- **비용 우선** → DeepSeek V4 (메인) + Grok 4.1 Fast (대용량)
- **균형** → Sonnet 4.6 (복잡) + Haiku 4.5 (단순) + DeepSeek V4 (대량)

가장 좋은 방법은 직접 본인의 에이전트 워크플로우에서 2~3개 모델을 **A/B 테스트**하는 것입니다. 같은 작업을 돌려보면 비용 대비 품질 차이가 명확하게 드러납니다.
